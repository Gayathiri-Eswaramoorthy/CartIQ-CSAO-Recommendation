Phase 3: TRANSFORMER MODEL
──────────────────────────

Architecture:

  Cart Input (8 items max, padded to 6)
    ↓
  Embedding Layer (128-dim, 2000 items)
    ↓
  Positional Encoding (preserves sequence order)
    ↓
  Transformer Encoder Block 1
    ├─ Multi-Head Attention (4 heads, 128-dim)
    ├─ Feed-Forward Network (256-dim hidden)
    └─ LayerNorm + Residual
    ↓
  Transformer Encoder Block 2
    └─ (same structure as Block 1)
    ↓
  Mean Pooling (across padded positions)
    ↓
  Feature Fusion
    ├─ Cart embedding (128)
    ├─ Candidate embedding (128)
    ├─ User features (7)
    ├─ Restaurant features (2)
    └─ Context features (2)
    ↓
  MLP Scoring Head
    ├─ FC(267 → 256) + ReLU + Dropout
    ├─ FC(256 → 128) + ReLU + Dropout
    ├─ FC(128 → 64) + ReLU + Dropout
    └─ FC(64 → 1) + Sigmoid
    ↓
  Output: Score ∈ [0, 1]

Parameters:
  • Total: ~850K
  • Trainable: ~850K
  • Model size: ~2 MB

Training:
  • Optimizer: Adam (LR=0.001, L2=1e-5)
  • Loss: Binary Cross Entropy
  • Early stopping: Patience=3 on Val NDCG@8
  • Typical convergence: 8-12 epochs

Performance (Test Set):

  • Precision@8: 0.0824 (+61% vs Popularity)

==================================================================================
KEY FEATURES & DESIGN DECISIONS
==================================================================================

1. ✅ Proper Temporal Split
   - Train/Val/Test strictly sequential (no future leakage)
   - Time-aware evaluation matches production scenario

2. ✅ Per-Cart Ranking Evaluation
   - Metrics computed per unique cart context
   - Realistic evaluation (not global ranking)
   - Real-world relevance: Top-K recommendations within order

3. ✅ Positional Encoding
   - Preserves sequence order of cart items
   - Items added first vs last have different influence
   - Attention masking for variable cart lengths

4. ✅ Feature Normalization
   - Statistics computed on TRAIN only
   - Applied to Val/Test to prevent leakage
   - Stable training and inference

5. ✅ Negative Sampling
   - Class imbalance handling
   - Efficient training on large dataset (5M rows)
   - 1 positive : 5 negatives ratio

6. ✅ Early Stopping
   - Monitor NDCG@8 on validation set
   - Patience=3 epochs prevents overfitting
   - Automatic convergence detection

7. ✅ Modular Architecture
   - 8 independent modules (data, model, train, eval, baselines, ablation, impact, main)
   - Easy to extend and debug
   - Clear separation of concerns

8. ✅ Reproducibility
   - Random seeds fixed (seed=42)
   - Deterministic data splits
   - Documented hyperparameters
